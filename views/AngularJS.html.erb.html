1) Introduction
"A client-side JavaScript Framework for adding interactivity to HTML"

How it works:
  • Initially you send a request to the server and you get a response with Webpage & Assets
  • However, when the user clicks the link it only requests the information the user needs to refresh the page
 But how do we tell the html when to reload? Directives!

A Directive is a marker on a HTML tag that tells Angular to run or reference some Javascript Code.
ex: 
index.html: <body ng-controller="StoreController"> This references a function written in app.'s called StoreController.
app.js: function StoreController(){ alert('Welcome, Henok!'); };

 Getting Started:
INDEX.HTML
     <html>
          <head>
               <link rel="stylesheet" type="text/css" href="bootstrap.min.css" />
          </head>
          <body>
               <script type="text/javascript" src="angular.min.js"></script>
          </body>
     </html> 

Modules
  • Where we write pieces of Angular application.
  • Where we define dependencies
EX: app.js: car app = angular.module('store', [ ]);

So if we include it in the html, it will treat the html inside of the element as part of the angular app. 

Expressions:
  • Let you insert dynamic values into your HTML
EX: index.html: <p> I am {{4+6}} </p> -> <p> I am 10 </p>

So…

INDEX.HTML
     <html ng-app="store">
          <head>
               <link rel="stylesheet" type="text/css" href="bootstrap.min.css" />
          </head>
          <body>
               <script type="text/javascript" src="angular.min.js"></script>
               <p> {{"hello" + "you"}} </p>
          </body>
     </html>

NgShow Directive

Looking at the app.js file, we want to add a new key/value to our gem 

APP.JS
(function(){

     var app = angular.module('store', [ ]);
     app.controller('StoreController', function(){
          this.product = gem;
     });
     var gem = {
          name: 'Dodecahedron',
          price: 2.95,
          description: '. . . ',
          canPurchase: false
     }
})( );

And we want a button to show up on the page if canPurchase is ONLY true. HOW WOULD WE DO THIS?

INDEX.HTML
 <body ng-controller="StoreController as store">
     <div>
          <h1> {{store.product.name}} </h1>
          <h2> ${{store.product.price}} </h2>
          <p> {{store.product.description}} </p>
          <button ng-show="store.product.canPurchase"> Add to Cart </button>
     </div>
     <script type = "text/javascript" src="angular.min.js"></script>
     <script type = "text/javascript" src="app.js"></script>
</body>

What this does is only display the element if the value of the expression is TRUE

And if the item is sold out, we don't want to show it so we can also use no-show for that.

APP.JS
(function(){
     var app = angular.module('store', [ ]);
     app.controller('StoreController', function(){
          this.product = gem;
     });
     var gem = {
          name: 'Dodecahedron',
          price: 2.95,
          description: '. . . ',
          canPurchase: false,
          soldOut: true
     }
})( );

INDEX.HTML
<body ng-controller="StoreController as store">
     <div ng-show="!store.product.soldOut">
          <h1> {{store.product.name}} </h1>
          <h2> ${{store.product.price}} </h2>
          <p> {{store.product.description}} </p>
          <button ng-show="store.product.canPurchase"> Add to Cart </button>
     </div>
     <script type = "text/javascript" src="angular.min.js"></script>
     <script type = "text/javascript" src="app.js"></script>
</body>

OR

<body ng-controller="StoreController as store">
     <div ng-hide="store.product.soldOut">
          <h1> {{store.product.name}} </h1>
          <h2> ${{store.product.price}} </h2>
          <p> {{store.product.description}} </p>
          <button ng-show="store.product.canPurchase"> Add to Cart </button>
     </div>
     <script type = "text/javascript" src="angular.min.js"></script>
     <script type = "text/javascript" src="app.js"></script>
</body>

The hide and show do the same thing, hide is just more readable.

2) Filters, Directives, and Cleaner Code

A little review…

ng-app: Attach the Application Module to the page
<html ng-app = "store">

ng-controller: Attach a Controller function to the page
<body ng-controller = "StoreController as store">

ng-show/ng-hide: Display a section based on an Expression
<h1 ng-show = "name"> Hello, {{name}}! </h1>

ng-repeat: Repeat a section for each item in an Array
<li ng-repeat="product in store.products"> {{product.name}} </li>


Our First Filter

<em  class="pull-right"> {{product.price | currency }} </em>

The currency formats this into currency because the | (pipe) sends the output into the currency filter.

Example other filters: date, upper/lower case, limitTo (limit number of characters or limit number of products by passing in "product in store.products | limitTo:3"), orderBy (-price = descending)

ng-src lets you dynamically load images and other things

But, what is really crazy is how the input tag's classes work in the background with validity!!!

The source before you start typing has class ng-pristine and ng-invalid which means that the field is empty and also invalid. As you start typing, the field takes on the class ng-dirty which means there is information with in but it is still invalid. After you finish typing with a valid email, the ng-invalid turns to ng-valid. What?!! Crazy. Wow.

But now we run into an interesting idea, we could use ng-include to pull this information but why not instead use a custom directive. Confused as to what it is? Look below. 



A custom directive basically allows you to write HTML, which typically only tells you about the structure, that expresses the behavior of your application. WOW!!!

Here is an example of custom directives in action.


So, how do you write one of these so-called custom directives?

What you do is you go into your javascript file and make, just like a controller, a directive using app.directive which would take the camelCase of the html dash separated tag.



Now the second part of the directive's parameters is an anonymous function which returns a directive definition object (Less fancy jargon: a configuration on how the directive will work).



In a basic example, we would just want this directive to load up the code we put into our product-title.html file before. In order to do this, we need to define what TYPE of directive it is, and the url that it needs to access.

We use type E since we are declaring a new HTML element. The templateUrl is just the url that we want to load onto the page. 



Now, the directive is an Element directive so some of you may be thinking "Why don't we just use a self-closing tag?" Well, some browsers have problems with self closing tags on custom elements so in order to be safe we shy away from that in Angular. 



But element directives aren't the only kinds. There are also Attribute Directives where you specify the name as an attribute within another element. However, its best to use element directives for UI and attribute directives when you add behaviors to HTML.
